## المقدمة

في ظل التوسع الحضري وزيادة عدد السكان، أصبحت إدارة النفايات تحديًا بيئيًا كبيرًا في المدن المتنامية كبريدة. يوفر هذا النظام حلاً رقمياً متكاملاً يعتمد على البيانات الفورية والذكاء الاصطناعي لتحسين كفاءة إدارة النفايات وتقليل أثرها البيئي، بما يتوافق مع مستهدفات رؤية 2030.

## الأثر البيئي

| المؤشر | كيف يساهم النظام |
|--------|-----------------|
| تقليل رحلات الشفط الفارغة | أوامر الشفط تنطلق فقط عند اقتراب الحاويات من الامتلاء (≥ 85%) |
| خفض استهلاك الوقود | تحسين مسارات الشفط يعني رحلات أقل وانبعاثات CO₂ أدنى |
| منع حرائق النفايات | نظام إنذار الحريق يكتشف ارتفاع درجات الحرارة والغازات مبكراً |
| دعم إعادة التدوير | تصنيف النفايات حسب النوع يُسهّل فرزها وإعادة تدويرها |
| قرارات مبنية على البيانات | لوحة الإدارة العليا تتبع هدف "صفر نفايات 2030" |
| بيئة حضرية أنظف | استجابة سريعة لبلاغات المواطنين تمنع تراكم النفايات |

## المميزات الكاملة

### صفحة تسجيل الدخول — تصميم حكومي
- تخطيط عمودين: يمين (هوية + شرح) / يسار (نموذج الدخول)
- شريط حكومي علوي يحمل هوية المملكة العربية السعودية
- شعار مركّب فاخر (`AppLogo`) موحّد عبر جميع البوابات
- شرح مختصر عن المنصة وارتباطها برؤية 2030
- إظهار/إخفاء كلمة المرور في جميع الحقول
- تذييل أمني يُظهر تشفير TLS 1.3
- دعم كامل للجوال (تخطيط عمود واحد تلقائياً)
- استعادة كلمة المرور عبر البريد الإلكتروني

### إدارة المحطات
- عرض جميع المحطات مع مؤشرات الامتلاء والضغط والنفايات اليومية
- إضافة وتعديل وحذف المحطات
- فلترة المحطات حسب الحالة (طبيعي / تحذير / حرج)
- ترتيب المحطات حسب مستوى الامتلاء أو عدد الحاويات

### إدارة الحاويات
- CRUD كامل للحاويات داخل كل محطة
- مؤشرات خطر مرمّزة بالألوان مع تأثير توهّج للحالات الحرجة
- ترتيب الحاويات حسب درجة الخطر

### وحدة التحكم بالشفط
- إصدار أوامر شفط فورية أو مجدولة أو متكررة يومياً
- تحديد نوع العملية: من حاوية إلى محطة، أو من محطة إلى المركزية
- متابعة الأوامر النشطة والسجل التاريخي في الوقت الفعلي

### نظام التنبيهات — محسّن
- زر جرس قابل للضغط في صفحة الموظف يفتح dropdown تنبيهات حقيقياً
- عرض قائمة التنبيهات مع كود لوني (أحمر / أصفر / أخضر)
- عداد الحالات الحرجة فوق أيقونة الجرس
- تنبيهات تلقائية عند تجاوز حدود الامتلاء
- إنذارات حريق مع تحليل درجة الحرارة ومستوى الغاز

### بوابة الإدارة العليا
- تحليلات مالية وأداء ربعي
- مقارنة أداء الأحياء والمقارنة المعيارية الوطنية
- سيناريوهات القرار مع تحليل العائد والمخاطر
- الأهداف الاستراتيجية وتوافقها مع رؤية 2030
- انتهاء الجلسة تلقائياً بعد 30 دقيقة من الخمول

### بوابة المواطن
- تقديم طلبات حاويات جديدة والإبلاغ عن مشاكل
- متابعة حالة الطلبات والبلاغات بشكل فوري
- عرض الحاويات المعتمدة بعد موافقة الموظف

### إدارة الطلبات والبلاغات
- قبول أو رفض طلبات المواطنين مع إمكانية إضافة رد
- تحديث حالة البلاغات (تم الحل / تحت المتابعة / مرفوض)
- التحديث ينعكس فوراً على حساب المواطن

### الرسوم البيانية والتحليلات — محسّنة للجوال
- مكوّن `ChartBox` مخصص يستخدم `ResizeObserver` لقياس العرض الحقيقي
- يحل مشكلة اختفاء المخططات على الجوال بشكل نهائي
- 34 مخططاً تعمل على جميع الأجهزة (جوال / تابلت / لابتوب)
- مخططات مساحية وشريطية ودائرية وراداريه (Recharts)
- بيانات أداء أسبوعية وشهرية وربعية

### الأيقونات — Lucide Icons
- استبدال جميع الإيموجي بأيقونات Lucide React احترافية
- تغطية كاملة: بوابة الموظف، بوابة المواطن، بوابة الإدارة العليا، صفحة الدخول
- أيقونات موحّدة ومتسقة بصرياً عبر كامل التطبيق

## هيكل قاعدة البيانات (Firestore)

```
Firestore
 ├── stations          # المحطات
 │    └── { name, district, fillLevel, pressure, dailyWaste, wasteType }
 ├── containers        # الحاويات
 │    └── { name, fillLevel, stationId, district }
 ├── sensors           # المستشعرات
 │    └── { stationId, containerId, type, value, unit, lastReading }
 ├── alerts            # التنبيهات
 │    └── { stationId, stationName, type, message, severity, resolved }
 ├── fire_alerts       # إنذارات الحريق
 │    └── { stationId, temperature, gasLevel, smokeDetected, riskLevel, resolved }
 ├── suctionJobs       # أوامر الشفط
 │    └── { type, scheduleType, scheduledAt, status, fromId, toId }
 ├── requests          # طلبات الحاويات
 │    └── { userId, userName, district, address, binType, status, response }
 ├── citizen_reports   # بلاغات المواطنين
 │    └── { userId, userName, district, type, description, status, response }
 ├── districts         # الأحياء
 │    └── { name, city, fillLevel, wasteTotal, performance, stationCount }
 ├── users             # المستخدمون
 │    └── { uid, email, fullName, role, district, status, createdAt }
 └── analytics         # التحليلات (مشتقة)
      ├── weekly        # بيانات أسبوعية
      ├── monthly       # بيانات شهرية
      ├── districts_perf # أداء الأحياء
      └── station_history # سجل يومي للمحطات
```

## تشغيل المشروع محلياً

### المتطلبات
- Node.js 18+
- حساب Firebase مع Firestore و Authentication مفعّلَيْن

### خطوات التشغيل

```bash
# 1. استنساخ المستودع
git clone https://github.com/your-username/smart-waste-mis.git
cd smart-waste-mis

# 2. تثبيت الحزم
npm install

# 3. إعداد Firebase — عدّل src/firebase.js بإعدادات مشروعك
```

```js
// src/firebase.js
import { initializeApp } from "firebase/app";
import { getAuth } from "firebase/auth";
import { getFirestore } from "firebase/firestore";

const firebaseConfig = {
  apiKey:            "YOUR_API_KEY",
  authDomain:        "YOUR_PROJECT.firebaseapp.com",
  projectId:         "YOUR_PROJECT_ID",
  storageBucket:     "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId:             "YOUR_APP_ID",
};

const app = initializeApp(firebaseConfig);
export const auth = getAuth(app);
export const db   = getFirestore(app);
```

```bash
# 4. تشغيل خادم التطوير
npm run dev
```

افتح المتصفح على: `http://localhost:5173`

> **تهيئة قاعدة البيانات:** بعد تسجيل الدخول كموظف، اذهب إلى **الإعدادات** واضغط **"تهيئة Collections الناقصة"** لإنشاء البيانات الأولية.

## النشر على Vercel

npm install -g vercel
