1 نظرة عامة على النظام

Smart Waste MIS هو نظام متكامل لإدارة البنية التحتية لشفط النفايات، مبني بتقنيات حديثة لتوفير بيانات فورية (Realtime) لمستويات الامتلاء والضغط وحالة الحاويات. يخدم النظام ثلاث فئات من المستخدمين: الموظفون، الإدارة العليا، والمواطنون.

2 البنية المعمارية العامة

يتكون النظام من ثلاث طبقات أساسية تعمل بشكل متكامل:

الشكل 1 — البنية المعمارية ثلاثية الطبقات للنظام

شرح الطبقات

طبقة العرض (Presentation Layer): مبنية بـ React 18 مع Vite 5 كأداة بناء. تحتوي على ثلاث بوابات منفصلة يتم التوجيه إليها تلقائياً حسب دور المستخدم بعد تسجيل الدخول. التصميم متجاوب بالكامل مع دعم الأجهزة المحمولة.

طبقة الخدمات (Services Layer): تعتمد على Firebase كمنصة خلفية (BaaS). تشمل المصادقة عبر Firebase Auth، قاعدة بيانات فورية عبر Cloud Firestore، تحليلات بيانية عبر Recharts، وتصدير البيانات عبر SheetJS.

طبقة البيانات (Data Layer): تتكون من 9 مجموعات (Collections) في Firestore تخزن جميع بيانات النظام من محطات ومستشعرات وتنبيهات وطلبات.

🟡
تحذير
fillLevel ≥ 60%

7 الوحدات والمكونات الرئيسية

🏭
إدارة المحطات

عرض جميع المحطات مع مؤشرات الامتلاء والضغط. إضافة وتعديل المحطات مع فلترة حسب الحالة وترتيب حسب الامتلاء.

🗑️
إدارة الحاويات

CRUD كامل للحاويات داخل كل محطة. مؤشرات خطر مرمّزة بالألوان مع تأثير توهّج للحالات الحرجة.

🚿
وحدة التحكم بالشفط

إصدار أوامر شفط فورية أو مجدولة أو متكررة. تحديد نوع العملية مع متابعة الأوامر النشطة والسجل التاريخي.

🔔
نظام التنبيهات

تنبيهات تلقائية عند تجاوز حدود الامتلاء. إنذارات حريق مع تحليل درجة الحرارة والغاز. تصنيف ثلاثي.

📊
التحليلات والرسوم البيانية

مخططات مساحية وشريطية ودائرية عبر Recharts. بيانات أداء أسبوعية وشهرية وربعية.

📬
إدارة الطلبات والبلاغات

قبول أو رفض طلبات المواطنين مع إمكانية إضافة رد. التحديث ينعكس فوراً على حساب المواطن.

8 هيكل الملفات

📁 src/ ├── App.jsx # التطبيق الكامل (مكونات + صفحات + منطق) ├── firebase.js # إعداد وتهيئة Firebase ├── firestoreService.js # طبقة Firestore (Hooks + CRUD) ├── exportAllDataToExcel.js # تصدير كامل قاعدة البيانات إلى Excel ├── seedFirestore.js # تهيئة البيانات التجريبية الأولية └── index.css # الأنماط العامة والتجاوب مع الجوال

النظام مبني بنمط Single File Architecture حيث يتضمن ملف App.jsx جميع المكونات والصفحات والمنطق في ملف واحد، بينما يتم فصل خدمات Firebase وطبقة البيانات في ملفات مستقلة.
